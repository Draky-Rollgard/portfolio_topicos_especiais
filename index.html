A Unidade

Introdução Técnica da Disciplina

O desenvolvimento de software corporativo baseia-se na consolidação de arquiteturas robustas e bem estruturadas. No contexto de sistemas distribuídos, a adoção da Arquitetura Orientada a Serviços (SOA) permite que componentes independentes colaborem de forma descentralizada através de Web Services e do modelo arquitetural REST.

As aplicações web modernas são estruturadas em camadas funcionais: Front-End, Back-End e Banco de Dados. Toda a comunicação síncrona entre o cliente e o servidor apoia-se no protocolo HTTP, seguindo o fluxo de Requisição (Request) e Resposta (Response).

Spring Boot

Autoconfiguração e Propriedades

O ecossistema Spring consolida-se como a principal escolha técnica para o desenvolvimento corporativo Java. O Spring Boot introduz o princípio de convenção sobre configuração, disponibilizando um ambiente autoconfigurável e pronto para execução (production-ready).

  • Dependências (POM.xml): Gerenciamento com Apache Maven integrando web, jpa e lombok.
  • Application.properties: Arquivo central para definir o acoplamento do sistema ao banco de dados e ditar o comportamento dinâmico do Hibernate.
# Configurações do Mecanismo de ORM (JPA / Hibernate)
spring.jpa.database-platform=org.hibernate.dialect.MySQLDialect
spring.jpa.show-sql=true
spring.jpa.hibernate.ddl-auto=update

# Indentamento visual e formatação de logs SQL
spring.jpa.properties.hibernate.format_sql=true

Padrão MVC e APIs

Model-View-Controller e Endpoints

O padrão de arquitetura MVC organiza a aplicação de modo a desacoplar a lógica de apresentação da lógica de controle e dos dados. A camada de controle atua como a interface de entrada que mapeia os endpoints expostos, aciona o processamento das regras de negócio e formata a resposta técnica em JSON.

  • @RestController: Indica que as respostas dos métodos devem ser escritas diretamente no corpo da mensagem HTTP.
  • Mapeamento: Anotações como @GetMapping e @PostMapping definem os verbos HTTP correspondentes.
  • Parâmetros: Uso de @RequestBody, @PathVariable e @RequestParam para extrair dados da requisição.
@RestController
@RequestMapping("/usuarios")
public class UsuarioController {
    
    @PostMapping("/save")
    public Usuario save(@RequestBody Usuario usuario) {
        return repository.save(usuario);
    }
    
    @GetMapping("/listarTodos")
    public List<Usuario> listarTodos() {
        return repository.findAll();
    }
}

Persistência de Dados

Java Persistence API (JPA)

O mecanismo de persistência orientada a objetos utiliza a especificação JPA em conjunto com o Hibernate. O propósito é o Mapeamento Objeto-Relacional (ORM), eliminando o descompasso de impedância ao converter tabelas e tuplas relacionais de bancos de dados diretamente em classes e atributos no paradigma orientado a objetos.

  • @Entity e @Data: A classe de modelo representa a estrutura estática da tabela, e o Lombok elimina métodos repetitivos de encapsulamento.
  • JpaRepository: Interface base que introduz rotinas padrão de manipulação de dados (CRUD).
  • JPQL: Consultas customizadas com suporte a parâmetros nomeados, funções de agregação (SUM, COUNT) e operadores lógicos e relacionais.
public interface UsuarioRepository extends JpaRepository<Usuario, Long> {
    
    // JPQL Personalizado com LIKE
    @Query("SELECT u FROM Usuario u WHERE u.nome LIKE CONCAT('%', :nome, '%')")
    List<Usuario> buscarContem(@Param("nome") String nome);

    // Funções de Agregação (Analytics)
    @Query("SELECT COUNT(u.nome) FROM Usuario u")
    Long contarNomes();
}

Testes e Rotas (Postman)

Validação da API REST

Para garantir o funcionamento adequado da nossa API, utilizamos a ferramenta Postman. Ela simula o papel de um cliente (como o navegador ou um Front-End) permitindo disparar requisições HTTP diretamente para o servidor Spring Boot (ex: http://localhost:8081/).

  • Criação e Atualização: Rotas com POST e PUT recebem objetos JSON no corpo (Body) para popular ou alterar o banco.
  • Consultas Detalhadas: Testamos as rotas GET para recuperar o JSON com a lista de usuários, validar as buscas customizadas com LIKE e a extração por @PathVariable.
  • Respostas e Status: Em cada teste, confirmamos se o servidor retornava o Status Code 200 (OK) e os dados esperados.
// Simulação de Requisição via Postman
URL: http://localhost:8081/save
Método: POST
Headers: Content-Type: application/json
Body (Raw/JSON):
{
  "usuario": "marcio",
  "senha": "123"
}

// Resposta Recebida do Servidor:
{
  "idUsuario": 2,
  "usuario": "marcio",
  "senha": "123"
}
